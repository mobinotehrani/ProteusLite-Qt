# بخش هفتم: قطعات پیشرفته، مبدل‌ها و میکروکنترلر

[بازگشت به README اصلی](README.md)

در بخش هفتم، پروژه از شبیه‌سازی قطعات پایه عبور می‌کند و چند قطعه پیشرفته به مدل مدار اضافه می‌شود. هدف این مرحله ساخت یک شبیه‌ساز صنعتی و دقیق برای میکروکنترلرها نیست؛ هدف این است که ساختارهای اصلی مانند تبدیل آنالوگ و دیجیتال، اجرای Firmware، رجیسترها، حافظه، پورت‌های ورودی و خروجی و چند Peripheral رایج به شکل شیءگرا و قابل فهم پیاده‌سازی شوند.

شاخه پیشنهادی این بخش:

```text
feature/advanced-components
```

## قابلیت‌های اضافه‌شده

- ADC ایده‌آل با رزولوشن قابل تنظیم
- DAC ایده‌آل با رزولوشن قابل تنظیم
- تأخیر تبدیل برای ADC و DAC
- نگه داشتن آخرین خروجی معتبر تا پایان تبدیل
- بارگذاری Firmware از فایل استاندارد Intel HEX
- بررسی ساختار خطوط، طول رکورد و Checksum فایل HEX
- حافظه Flash داخلی برای میکروکنترلر آموزشی
- Program Counter، رجیسترها و RAM داخلی به‌صورت کلاس‌های مستقل
- Fetch و Decode دستور در هر سیکل
- دستورهای `MOV`، `ADD`، `JMP`، `SETB` و `CLR`
- سه پورت هشت‌بیتی A، B و C
- تعیین جهت هر پایه به‌صورت ورودی یا خروجی
- حافظه خارجی RAM/EEPROM با باس آدرس، باس داده و کنترل‌های فعال‌پایین
- LCD کاراکتری 16x2 با فرمان‌های اصلی
- Keypad ماتریسی 4x4 با اتصال واقعی سطر و ستون هنگام فشردن کلید
- پنل `Section 7 Monitor` برای مشاهده وضعیت زنده قطعات این بخش

## 1. مبدل ADC

کلاس `AdcComponent` یک ورودی آنالوگ و دو پایه مرجع دارد:

```text
VIN
VREF-
VREF+
```

هنگام قرار دادن ADC روی بوم، یک پنجره کوچک تعداد بیت‌ها را می‌پرسد. تعداد پایه‌های خروجی دیجیتال بر اساس همین مقدار ساخته می‌شود. برای نمونه، ADC چهار بیتی خروجی‌های `D0` تا `D3` دارد و ADC هشت بیتی خروجی‌های `D0` تا `D7` را ایجاد می‌کند. در هر دو مبدل، `D0` بیت کم‌ارزش یا LSB است. مقدار Resolution بعداً از Properties نیز قابل تغییر است.

مدل تبدیل خطی است:

- اگر `VIN` برابر `VREF-` باشد، کد خروجی صفر است.
- اگر `VIN` برابر `VREF+` باشد، کد خروجی برابر `2^N - 1` است.
- مقادیر پایین‌تر از بازه روی صفر Saturate می‌شوند.
- مقادیر بالاتر از بازه روی بیشترین کد Saturate می‌شوند.
- هر بیت خروجی به‌صورت یک Pin مستقل، ولتاژ HIGH یا LOW تولید می‌کند.

ویژگی `Conversion delay` مدت زمان تبدیل را تعیین می‌کند. هنگام شروع تبدیل جدید، خروجی معتبر قبلی حفظ می‌شود و فقط پس از پایان تأخیر تغییر می‌کند.

## 2. مبدل DAC

هنگام قرار دادن DAC روی بوم نیز تعداد بیت‌ها از کاربر پرسیده می‌شود. کلاس `DacComponent` یک باس دیجیتال N بیتی، دو پایه مرجع و یک خروجی آنالوگ دارد:

```text
D0 ... D(N-1)
VREF-
VREF+
VOUT
```

کد دیجیتال به‌صورت خطی به بازه ولتاژ مرجع نگاشت می‌شود. کد صفر برابر `VREF-` و بیشترین کد برابر `VREF+` است. ورودی شناور یا Undefined باعث ایجاد هشدار می‌شود و خروجی قبلی تا رسیدن یک ورودی معتبر حفظ می‌شود.

## 3. بارگذاری Firmware

خواندن Firmware در کلاس جداگانه `IntelHexLoader` انجام می‌شود. این کلاس فایل را مستقیماً داخل مدل MCU تفسیر نمی‌کند؛ ابتدا آن را اعتبارسنجی می‌کند و سپس آرایه بایت‌های Flash را تحویل می‌دهد.

مواردی که بررسی می‌شوند:

- شروع هر رکورد با `:`
- زوج بودن تعداد کاراکترهای هگز
- هماهنگی Byte Count با طول رکورد
- معتبر بودن همه ارقام هگز
- معتبر بودن Checksum
- وجود رکورد پایان فایل `EOF`
- نبودن داده بعد از EOF
- قرار داشتن آدرس‌ها در محدوده Flash آموزشی 64 KiB

رکوردهای Data، EOF، Extended Segment Address و Extended Linear Address پشتیبانی می‌شوند. رکوردهای Start Address پذیرفته می‌شوند، اما در این مدل آموزشی اثری روی اجرای برنامه ندارند.

## 4. ساختار داخلی میکروکنترلر

پیاده‌سازی MCU به چند کلاس کوچک تقسیم شده است تا مسئولیت‌ها در یک کلاس بزرگ جمع نشوند.

### `ProgramCounter`

آدرس دستور بعدی را نگه می‌دارد و امکان Reset، Advance و پرش به یک آدرس جدید را فراهم می‌کند.

### `RegisterBank`

دو رجیستر اصلی مدل آموزشی را نگه می‌دارد:

- `A`: Accumulator
- `B`: General-purpose register

### `InternalRam`

RAM داخلی 256 بایتی است. خواندن و نوشتن فقط از طریق متدهای این کلاس انجام می‌شود و آرایه حافظه مستقیماً در اختیار سایر کلاس‌ها قرار نمی‌گیرد.

### `McuPort`

هر Port شامل سه مقدار است:

- Direction Mask
- Output Latch
- Input Sample

اگر بیت متناظر Direction برابر 1 باشد، پایه خروجی است و مقدار Output Latch را روی مدار قرار می‌دهد. اگر صفر باشد، پایه ورودی است و مقدار شبکه مدار خوانده می‌شود.

### `EducationalMcuCore`

Flash، Program Counter، رجیسترها، RAM و سه پورت در این کلاس کنار هم قرار می‌گیرند. متد `executeNext()` در هر سیکل یک Opcode را Fetch، Decode و اجرا می‌کند.

## 5. مجموعه دستورهای MCU آموزشی

این مدل، معماری واقعی AVR یا PIC را تقلید نمی‌کند. یک Instruction Set کوچک و مشخص برای نمایش چرخه Fetch/Decode/Execute ساخته شده است.

| Opcode | اندازه | دستور | توضیح |
|---|---:|---|---|
| `00` | 1 | `NOP` | بدون عملیات |
| `10 target imm` | 3 | `MOV target, immediate` | نوشتن مقدار ثابت در رجیستر، Port یا Direction |
| `11 dst src` | 3 | `MOV destination, source` | کپی مقدار بین مقصدهای معتبر |
| `20 target imm` | 3 | `ADD target, immediate` | جمع هشت‌بیتی با مقدار ثابت |
| `21 dst src` | 3 | `ADD destination, source` | جمع هشت‌بیتی دو مقصد |
| `30 low high` | 3 | `JMP address` | پرش به آدرس 16 بیتی Little Endian |
| `40 port bit` | 3 | `SETB` | یک کردن یک بیت از Output Latch |
| `41 port bit` | 3 | `CLR` | صفر کردن یک بیت از Output Latch |
| `50 address` | 2 | `STORE [address], A` | ذخیره Accumulator در RAM داخلی |
| `51 address` | 2 | `LOAD A, [address]` | خواندن RAM داخلی در Accumulator |
| `FF` | 1 | `HALT` | توقف اجرای Firmware |

کدهای مقصد برای `MOV` و `ADD`:

| کد | مقصد |
|---:|---|
| `00` | Accumulator A |
| `01` | Register B |
| `02` | Port A |
| `03` | Port B |
| `04` | Port C |
| `05` | Direction A |
| `06` | Direction B |
| `07` | Direction C |

در `SETB` و `CLR` شماره پورت صفر برای A، یک برای B و دو برای C است.

## 6. اتصال پورت‌های MCU به مدار

قطعه MCU دارای 24 پایه I/O است:

```text
PA0 ... PA7
PB0 ... PB7
PC0 ... PC7
```

دو پایه `VCC` و `GND` نیز برای تغذیه وجود دارند. گزینه `Require VCC and GND` مشخص می‌کند که اجرای Firmware به تغذیه معتبر وابسته باشد یا نه.

در هر Step شبیه‌سازی:

1. پایه‌های ورودی از شبکه مدار خوانده می‌شوند.
2. تعداد سیکل‌های لازم بر اساس زمان گذشته و فرکانس Instruction Clock محاسبه می‌شود.
3. دستورها اجرا می‌شوند.
4. پایه‌هایی که Direction آن‌ها خروجی است، مقدار جدید را روی مدار Drive می‌کنند.

برای جلوگیری از اجرای چندباره در Passهای تکراری حل شبکه، MCU فقط وقتی زمان شبیه‌سازی جلو رفته باشد دستور جدید اجرا می‌کند.

## 7. حافظه خارجی RAM/EEPROM

کلاس `ExternalMemoryComponent` از مدل MCU جداست و باس‌های مستقل دارد:

```text
A0 ... A7
D0 ... D7
/CS
/RD
/WR
```

کنترل‌ها Active Low هستند:

- خواندن: `/CS=0`، `/RD=0` و `/WR=1`
- نوشتن: `/CS=0`، `/RD=1` و `/WR=0`

در حالت خواندن، حافظه باس داده را Drive می‌کند. در حالت نوشتن، مقدار باس داده فقط یک بار در شروع سیکل نوشتن ذخیره می‌شود تا یک سطح ثابت روی `/WR` باعث نوشتن تکراری نشود.

## 8. LCD 16x2

کلاس `Lcd16x2Component` دارای پایه‌های زیر است:

```text
RS
RW
E
D0 ... D7
```

داده در لبه بالارونده `E` خوانده می‌شود. در این مدل، سیکل نوشتن پشتیبانی می‌شود و سیکل خواندن فقط هشدار آموزشی نمایش می‌دهد.

فرمان‌های اصلی:

- `0x01`: پاک کردن نمایشگر
- `0x02`: برگشت Cursor به خانه اول
- `0x80 | address`: تنظیم آدرس Cursor
- فرمان‌های Display On/Off
- فرمان Entry Mode برای افزایش یا کاهش Cursor

وقتی `RS=1` باشد، مقدار باس داده به‌عنوان کاراکتر در موقعیت Cursor نوشته می‌شود. نتیجه هم روی شکل LCD در بوم و هم در Monitor قابل مشاهده است.

## 9. Keypad ماتریسی 4x4

کلیدها با ترتیب زیر نمایش داده می‌شوند:

```text
1  2  3  A
4  5  6  B
7  8  9  C
*  0  #  D
```

هر کلید در زمان فشرده شدن دقیقاً یک سطر را به یک ستون متصل می‌کند. این اتصال از طریق `conductivePinPairs()` وارد مدل شبکه می‌شود؛ بنابراین MCU می‌تواند یک سطر یا ستون را Drive کند و طرف مقابل را به‌صورت ترتیبی بخواند.

با رها کردن ماوس، اتصال سطر و ستون حذف می‌شود. این رفتار به جای تولید مستقیم یک عدد، منطق واقعی Scan ماتریسی را شبیه‌سازی می‌کند.

## 10. رابط کاربری و Properties

برای Firmware یک نوع Property جدید با نام `FilePath` اضافه شده است. در پنجره Properties می‌توان مسیر را دستی وارد کرد یا با دکمه Browse فایل `.hex` را انتخاب کرد.

تغییر رزولوشن ADC یا DAC تعداد Pinها را تغییر می‌دهد. مانند گیت‌های چندورودی بخش ششم، کنترلر بخش پنجم اتصال‌های قدیمی همان قطعه را حذف می‌کند تا سیمی به Pin حذف‌شده باقی نماند.

قطعات پیشرفته شکل جداگانه دارند:

- ADC و DAC به شکل بلوک مبدل
- MCU به شکل IC چندپورتی
- حافظه با باس آدرس و داده
- LCD با دو خط زنده
- Keypad با 16 دکمه قابل کلیک

## 12. Firmware نمونه

فایل زیر داخل پوشه `examples` قرار دارد:

```text
examples/section07_blink.hex
```

این برنامه:

1. بیت صفر Port A را خروجی می‌کند.
2. `PA0` را HIGH می‌کند.
3. `PA0` را LOW می‌کند.
4. به مرحله دوم برمی‌گردد.

با Instruction Clock پایین، می‌توان تغییر وضعیت خروجی را در Monitor یا با LED متصل به `PA0` مشاهده کرد.

## 13. آزمون‌های پیشنهادی قبل از Commit

### ADC

1. ADC چهار بیتی را قرار دهید.
2. `VREF-` را به GND و `VREF+` را به 5 V وصل کنید.
3. چند مقدار ورودی بین صفر و پنج ولت امتحان کنید.
4. خروجی صفر، وسط بازه و کد 15 را بررسی کنید.
5. Conversion Delay را افزایش دهید و ثابت ماندن خروجی قبلی را ببینید.

### DAC

1. DAC چهار بیتی را قرار دهید.
2. مرجع‌ها را به صفر و پنج ولت وصل کنید.
3. کدهای `0000`، `1000` و `1111` را روی ورودی اعمال کنید.
4. ولتاژ خروجی و تأخیر تبدیل را بررسی کنید.

### MCU

1. MCU، منبع 5 V، GND، LED و مقاومت را قرار دهید.
2. `VCC` و `GND` را متصل کنید.
3. فایل `section07_blink.hex` را از Properties انتخاب کنید.
4. `PA0` را به LED و سپس به GND متصل کنید.
5. فرکانس را ابتدا 5 تا 20 Hz بگذارید تا تغییر قابل مشاهده باشد.
6. مقدار PC، آخرین دستور و Port A را در Monitor بررسی کنید.

### حافظه خارجی

1. پایه‌های کنترل را در حالت غیرفعال نگه دارید.
2. یک آدرس و داده مشخص بسازید.
3. سیکل نوشتن و سپس سیکل خواندن را اجرا کنید.
4. مقدار نشان‌داده‌شده در Monitor را مقایسه کنید.

### LCD

1. پایه‌های RS، RW، E و باس داده را به خروجی‌های MCU یا منابع دیجیتال وصل کنید.
2. فرمان Clear را با یک لبه بالارونده E ارسال کنید.
3. Cursor را تنظیم و یک کاراکتر بنویسید.
4. تغییر خط روی بوم را بررسی کنید.

### Keypad

1. سطرها و ستون‌ها را به Portهای MCU وصل کنید.
2. Firmware اسکن را اجرا کنید یا با منابع دیجیتال یک طرف ماتریس را Drive کنید.
3. یکی از کلیدها را نگه دارید.
4. اتصال سطر و ستون متناظر را بررسی کنید.
5. کلید را رها کنید و قطع شدن اتصال را ببینید.

## محدودیت‌های آگاهانه

- مدل ADC و DAC ایده‌آل است و نویز، Offset، INL و DNL ندارد.
- MCU یک معماری آموزشی اختصاصی است و فایل HEX کامپایل‌شده برای AVR، PIC یا ARM را به‌صورت واقعی اجرا نمی‌کند؛ فایل باید شامل Opcodeهای تعریف‌شده در همین مستند باشد.
- RAM/EEPROM فعلی باس آدرس هشت‌بیتی دارد و حداکثر 256 خانه را مستقیماً آدرس‌دهی می‌کند.
- مدل LCD روی رابط هشت‌بیتی تمرکز دارد و Busy Flag یا Timing دقیق سخت‌افزار واقعی را شبیه‌سازی نمی‌کند.
- شبیه‌سازی هنوز جایگزین حل‌گر حرفه‌ای مدار نیست و هدف آن نمایش ساختار شیءگرا و ارتباط اجزای پروژه است.

## جمع‌بندی

بخش هفتم چند مفهوم مهم سخت‌افزار و معماری کامپیوتر را به ساختار فعلی پروژه متصل می‌کند. ADC و DAC از همان شبکه ولتاژ استفاده می‌کنند، MCU ورودی‌ها را از شبکه می‌خواند و خروجی‌ها را به آن برمی‌گرداند، حافظه و LCD از باس‌های دیجیتال استفاده می‌کنند و Keypad از اتصال ماتریسی واقعی بهره می‌برد. این جداسازی باعث می‌شود هر قطعه رفتار خودش را داشته باشد و کنترلر شبیه‌سازی فقط هماهنگی شبکه را انجام دهد.
