# بخش سوم: بوم طراحی و محیط کار

[بازگشت به README اصلی](README.md)

در این مرحله، محیط اصلی طراحی مدار ساخته شد. هدف این بود که کاربر بعد از ایجاد یا بازکردن پروژه وارد فضایی شود که بتواند مثل یک صفحه شماتیک روی آن کار کند. این بخش بر پایه `QGraphicsView` و `QGraphicsScene` نوشته شده و اندازه صفحه از اطلاعات پروژه خوانده می‌شود.

شاخه مربوط به این بخش:

```text
canvas-workspace
```

## نتیجه این مرحله

بعد از اجرای برنامه و ساخت یک پروژه، یک صفحه طراحی در اختیار کاربر قرار می‌گیرد. Grid، مختصات و ابزارهای بزرگ‌نمایی روی همین صفحه فعال هستند و زیرساخت لازم برای قرار گرفتن قطعات و سیم‌ها در مراحل بعدی آماده شده است.

## قابلیت‌های پیاده‌سازی‌شده

- نمایش Grid منظم با فاصله ۲۰ پیکسل
- مشخص بودن خطوط اصلی Grid در هر پنج خانه
- نمایش محدوده صفحه طراحی
- نمایش مبدأ مختصات
- نمایش نقطه Snap زیر نشانگر ماوس
- نمایش مختصات عادی و مختصات Snap‌شده در Status Bar
- بزرگ‌نمایی و کوچک‌نمایی با چرخ ماوس
- Zoom با دکمه‌های نوار ابزار
- حرکت روی صفحه با دکمه وسط ماوس
- حرکت روی صفحه با `Space + Left Drag`
- روشن و خاموش کردن Grid
- روشن و خاموش کردن Snap
- برگشت به زوم ۱۰۰ درصد
- Fit کردن کامل صفحه داخل پنجره
- استفاده از اندازه انتخاب‌شده در زمان ساخت پروژه

## کلاس اصلی این بخش

### `CanvasView`

کلاس `CanvasView` از `QGraphicsView` ارث‌بری می‌کند و بیشتر رفتارهای مربوط به بوم در همین کلاس قرار گرفته‌اند. این کار باعث شده کد مربوط به Zoom، Pan، Grid و Snap داخل پنجره اصلی پخش نشود.

مهم‌ترین مسئولیت‌های این کلاس:

- مدیریت بزرگ‌نمایی
- مدیریت حرکت روی صفحه
- تبدیل موقعیت ماوس به مختصات Scene
- نزدیک کردن مختصات به Grid
- رسم پس‌زمینه و خطوط Grid
- ارسال مختصات برای نمایش در Status Bar

متد `snapToGrid` یکی از بخش‌های مهم این کلاس است. در مراحل بعدی همین متد برای قرار دادن قطعات، جابه‌جایی آن‌ها و تعیین نقاط مسیر سیم استفاده می‌شود.

## میانبرهای کاربردی

| عملکرد | میانبر |
|---|---|
| پروژه جدید | `Ctrl+N` |
| باز کردن پروژه | `Ctrl+O` |
| روشن یا خاموش کردن Grid | `Ctrl+G` |
| روشن یا خاموش کردن Snap | `Ctrl+Shift+G` |
| Zoom In | `Ctrl++` |
| Zoom Out | `Ctrl+-` |
| زوم ۱۰۰ درصد | `Ctrl+0` |
| نمایش کامل صفحه | `F` |
| حرکت روی صفحه | دکمه وسط ماوس یا `Space + Drag` |

## ارتباط با بخش‌های دیگر

اطلاعات اندازه صفحه از `ProjectDocument` دریافت می‌شود و `MainWindow` مسئول قرار دادن `CanvasView` در محیط اصلی برنامه است. در این مرحله هنوز قطعات به‌صورت کامل بازسازی نمی‌شوند و تمرکز فقط روی آماده‌سازی محیط طراحی بوده است.

## آزمون‌های انجام‌شده

برای بررسی این بخش موارد زیر تست شدند:

1. ساخت پروژه با اندازه‌های مختلف صفحه
2. تغییر Zoom با ماوس و نوار ابزار
3. حرکت روی صفحه با دو روش موجود
4. روشن و خاموش کردن Grid
5. روشن و خاموش کردن Snap
6. بررسی مختصات در Status Bar
7. برگشت به زوم ۱۰۰ درصد
8. Fit شدن کل صفحه در اندازه‌های مختلف پنجره

## جمع‌بندی

بخش سوم زیرساخت گرافیکی پروژه را آماده کرد. از این مرحله به بعد، قطعات، سیم‌ها و سایر آیتم‌های مدار می‌توانند روی یک بوم استاندارد و قابل کنترل نمایش داده شوند.
